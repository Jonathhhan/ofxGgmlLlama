# ofxGgmlLlama v1.0.1

## Compatibility

- openFrameworks: `of_v20260406_vs_64_release`
- minimum `ofxGgmlCore`: `1.0.1` / commit `2171e8b`
- `OFXGGML_LLAMA_VERSION_STRING`: `1.0.1`
- llama.cpp revision: `b9453` by default in `scripts/build-llama-server.*`

## Tested Runtime Backends

- CPU: build-script path covered by dry-run validation
- CUDA: build-script path supported; local full inference smoke not part of this RC pass
- Vulkan: build-script path supported; local full inference smoke not part of this RC pass
- Metal: build-script path supported; local full inference smoke not part of this Windows RC pass
- OpenCL: build-script path supported; local full inference smoke not part of this RC pass

## Highlights

- Concrete llama.cpp adapters now live in `ofxGgmlLlama`.
- `ofxGgmlCore` remains the dependency for shared text and embedding request/result APIs.
- Added independent Llama version metadata.
- Added migration notes for projects moving from Core-hosted llama adapters.
- Added release checklist, release policy, changelog, release-note template, and release-candidate gate.
- Added validation coverage for launch helper behavior, model discovery, release checklist commands, and adapter umbrella exposure.

## Breaking Changes

- Projects that included llama adapter headers from `ofxGgmlCore` must depend on `ofxGgmlLlama` and include `ofxGgmlLlama.h` or the adapter headers from this addon.

## Migration Notes

- See `docs/MIGRATION.md` for Core/Llama split include changes.
